Як зробити свій сайт зі звітами на GitHub Pages

Інструкція для колег. Результат: твої HTML-звіти відкриваються за посиланням у браузері (напр. https://ТВІЙ-ЛОГІН.github.io/reports/greek-house/), і кожному партнеру можна дати лише його папку.

Крок 0. Реєстрація на GitHub (одноразово)

  1. Зайди на github.com → Sign up.
  2. Створи акаунт (email, логін, пароль). Логін = майбутня адреса сайту (https://ЛОГІН.github.io/...).
  3. Підтверди пошту.

Як завантажити звіти через Cursor AI

AI сам створить репозиторій, завантажить файли, увімкне GitHub Pages і дасть готові посилання.

1. Створити токен (Personal Access Token)

  1. Відкрий github.com/settings/tokens/new (це Tokens → classic).
  2. Note: cursor-upload.
  3. Expiration: напр. 30 днів.
  4. Scopes: постав галочку repo (весь блок).
  5. Generate token → скопіюй (починається з ghp_...).
⚠️ Токен — це секрет. Нікому не пересилай, не зберігай у файлах. Після роботи можна відкликати на github.com/settings/tokens.

2. Дати завдання AI у Cursor

Напиши приблизно так:

«Ось мій токен ghp_.... Створи публічний репозиторій reports, поклади ці HTML-файли у папки greek-house/, salateira/ тощо, увімкни GitHub Pages і дай мені посилання на кожну папку.»

AI зробить усе сам і поверне готові посилання.

Приватність — важливо розуміти

  • Сайт публічний: хто має посилання, той відкриє. Логінів/паролів немає.
  • Партнери не бачать одне одного, якщо ти:
    • НЕ робиш головну сторінку зі списком усіх (корінь /reports/ тоді віддає 404);
    • даєш кожному лише посилання на його папку.
  • Захист тримається на тому, що посилання важко вгадати. Для комерційних даних це «захист через неочевидність».
  • Потрібен справжній захист паролем? Тоді використовуй Netlify (безкоштовно) замість GitHub Pages — там можна поставити пароль на сайт.

Службова сторінка «всі мережі» (для себе / керівника)

Можна створити файл із випадковою назвою (напр. admin-3e561cef51.html) у корені репозиторію зі списком усіх папок. Оскільки назва невгадувана і ніде не залінкована, її мають лише ті, кому ти надіслала. Партнерам не давати.

Шаблон index.html (для папки партнера)

Скопіюй у файл index.html усередині папки партнера, заміни назву й посилання на звіти:

<!DOCTYPE html>
<html lang="uk">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Назва мережі — Reports</title>
  <style>
    body { font-family: -apple-system, Segoe UI, Roboto, sans-serif; max-width: 720px; margin: 60px auto; padding: 0 20px; color: #1a1a1a; }
    h1 { font-size: 28px; }
    ul { list-style: none; padding: 0; }
    li { margin: 12px 0; }
    a { display: inline-block; padding: 14px 18px; background: #f4f4f5; border-radius: 10px; text-decoration: none; color: #0969da; font-weight: 600; }
    a:hover { background: #e6e6e9; }
  </style>
</head>
<body>
  <h1>Назва мережі — Reports</h1>
  <ul>
    <li><a href="звіт1.html">Назва звіту 1</a></li>
    <li><a href="звіт2.html">Назва звіту 2</a></li>
  </ul>
</body>
</html>

Часті питання

Скільки коштує? — Безкоштовно (GitHub Pages на публічному репозиторії).

Скільки чекати після завантаження? — Зазвичай 1–2 хвилини, поки сайт «збереться».

Файл не відкривається (404)? — Перевір, що: (1) Pages увімкнено; (2) шлях/назва файлу точні (з урахуванням великих/малих літер); (3) минуло кілька хвилин.

Можна свій домен? — Так, у Settings → Pages → Custom domain, але це вже додаткове налаштування.
